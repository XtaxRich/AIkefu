# 06 Artifacts and Results

## 目的
本文件定义实验产物、评估结果和中间资产应如何沉淀，避免“做过实验但仓库里没有留下可复用结果”。

## 应沉淀的产物类型
### 1. 设计类产物
- 模块说明
- 输入输出契约
- 失败模式说明
- 版本差异说明

### 2. 运行类产物
- 代表性 trace 样例
- replay 输入样例
- 决策输出样例
- 错误样例

### 3. 评估类产物
- 样例评分结果
- 指标统计摘要
- 错误聚类摘要
- baseline 对比结果

## 结果沉淀规则
1. 结果必须和对应模块绑定。  
2. 结果必须能追溯到输入 case。  
3. 结果必须能区分“离线结果”和“线上结果”。  
4. 结果摘要必须包含已知限制，不能只写正向结论。

## 禁止事项
- 只在聊天里说实验结论，不入仓库
- 只保留截图，不保留结构化结果
- 只写平均指标，不保留失败样例
- 把一次偶然跑通当成稳定结果

## 推荐做法
每个重要实验都至少留下：
- 输入说明
- 输出说明
- 一份 trace 样例
- 一份结果摘要
- 一份限制说明
